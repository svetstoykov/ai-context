# Architecture Decision Records

## What It Is

A reusable rule for documenting significant architectural decisions as short Architecture Decision Records.

## When To Use

Use this when an agent proposes or makes an architectural choice that affects dependencies, contracts, data models, infrastructure, performance strategy, security approach, or long-term maintainability.

## How To Use

Copy this folder into a repository rules directory, or reference `RULE.md` directly from project instructions. Apply the rule before or alongside implementation so the decision context and tradeoffs are preserved.

## Contents

- `RULE.md` - agent-facing ADR trigger conditions, file location, format, and writing rules.
