# Task Writing

## What It Is

A reusable skill for turning feature ideas, bug reports, and change requests into clear development tasks.

## When To Use

Use this when an agent is asked to create a task, write a ticket, define work items, plan a feature, break down work, or formalize a requested change into backlog-ready documentation.

## How To Use

Copy this folder into an agent skill directory, or reference `SKILL.md` directly from project instructions. Apply the skill before writing tasks so each task has clear scope, acceptance criteria, dependencies, and a definition of done.

## Contents

- `SKILL.md` - agent-facing instructions and a reusable task template.
